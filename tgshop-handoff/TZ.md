# ТЗ — Лендинг TGShop Chat (UZ)

## 1. Назначение
Маркетинговый лендинг продукта **TGShop Chat** для рынка Узбекистана.
Целевой домен — поддомен в зоне `uz.tgshop.io` или отдельный.
Язык — русский (с возможностью добавить `uz` локаль через переключатель в header).

## 2. Технические требования

### 2.1. Стек (рекомендация)
- **HTML5 + CSS3 + минимум JS** — это статический лендинг
- **Tailwind CSS** или плэйн CSS — на выбор
- **Адаптив через CSS media queries** — не JS, чтобы не было FOUC
- Без зависимостей вроде React (статика быстрее грузится для маркетинга)
- Если в команде уже Next.js — можно как статическая страница без гидрации

### 2.2. Производительность
- LCP < 2.5s
- CLS < 0.1
- Total page size < 1MB (без видео)
- Лого / мокапы — SVG где возможно, остальное WebP с lazy-loading
- Шрифт — `SF Pro` (Apple system) или fallback `-apple-system, BlinkMacSystemFont, "Inter", sans-serif`
- Подключить через `font-display: swap`

### 2.3. SEO
- Title: `TGShop Chat — Telegram-магазин и CRM для бизнеса в Узбекистане`
- Description: `Запустите магазин в Telegram за 5 минут без сайта и разработчиков. Каталог, оплата через Click и Payme, CRM в одном кабинете.`
- Open Graph + Twitter Cards — обязательно
- `lang="ru"` + `hreflang` для `ru` и `uz` версий
- Sitemap.xml + robots.txt
- Schema.org Product/SoftwareApplication разметка для тарифов

### 2.4. Аналитика
- Yandex Metrika (счётчик из текущего сайта tgshop.io)
- Telegram Ads pixel (если используется)
- Цели: клики по CTA «Зарегистрироваться» и «Войти», скролл-глубина, время на странице

## 5. Header

**Background:** белый `#FFFFFF`
**Высота:** 72px desktop, 72px tablet, 60px mobile
**Контент:**
- Слева: лого `TGShop` (+ small подпись `Chat` голубая)
- Центр: навигация (только desktop/tablet): `База знаний / Тарифы / Помощь / Решения`
- Справа:
  - Desktop/Tablet: кнопки `Войти` (outline) + `Зарегистрироваться` (primary blue)
  - Mobile: переключатель языка `🌐 RU ▾` + бургер-меню

**Бургер-меню (Mobile)** при клике открывает оверлей:
- Лого TGShop сверху + крестик ✕
- Список ссылок (4 пункта nav)
- Кнопки `Зарегистрироваться` (primary) + `Войти` (secondary)

Все ссылки — см. файл `content-and-links.md`.

## 6. Footer

**Текст:** белый для заголовков блоков, светло-серый `#A6AFBE` для ссылок
**4 колонки** на Desktop/Tablet, **столбик** на Mobile:
1. Бренд + описание
2. Полезные материалы
3. Решения
4. Поддержка

Внизу: `© 2026 TGShop. Все права защищены.`

Подробные ссылки — см. `content-and-links.md`.

## 7. Hero — особенности

- **3 floating badges** вокруг продуктового мокапа (десктоп вид):
  - `Запуск за 15 минут / без программистов` (большая, повёрнута -6°)
  - `+37 новых заказов сегодня` (зелёная, +3°)
  - `12 новых заявок в чатах` (нейтральная, 0°)
- `clipsContent: false` на родительском контейнере чтобы badges могли «вылетать» за границы
- На Mobile — упрощённая версия (1-2 floating badge максимум, остальное скрыто)

## 8. Калькулятор (Price section)

**Логика:** статический, БЕЗ ввода данных (на лендинге показано как пример). Реальная интерактивность — на другой странице.
- Bar chart 30 столбцов с градиентом красного — анимация роста (опционально)
- Финальная цифра «45 000 000 soʻm» — большая, красная
- Подпись «Средний магазин · 30 рабочих дней»

## 9. Тарифы

3 карточки одинакового размера. На карточке «Бизнес» badge **«Популярный»**.

Каждая карточка: название → цена `XXX 000 soʻm/мес` → старая цена (зачёркнутая) → лимиты → список фич (✓ зелёные + ✕ красные) → кнопка `Начать бесплатно` (primary).

Цены актуальны, не менять без согласования.

## 10. FAQ

Стандартный аккордеон с `+` иконкой. При клике — раскрытие.
В вёрстке: `<details><summary>` или JS-based, на выбор.

## 12. Анимации (опционально, можно отложить)

- Hero badges — лёгкое float (3-4s loop, 5-10px по Y)
- При скролле — fade-in секций (Intersection Observer)
- Hover на cards — небольшой lift (translateY(-4px) + shadow)
- Bar chart — staggered grow на view (delay по индексу)
